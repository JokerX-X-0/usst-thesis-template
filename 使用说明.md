# 上海理工大学硕士/博士学位论文 LaTeX 模板使用说明

本文档说明如何使用当前文件夹中的上海理工大学硕士/博士学位论文 LaTeX 模板。普通写作者通常只需要修改 `main.tex`、`chapters/`、`figures/` 和 `references.bib`，不要直接修改 `ussthesis.cls`。只有学校格式规范变化，或者需要调整封面、页眉、目录、标题格式等全局样式时，才考虑修改 `ussthesis.cls`。

## 1. 先了解模板如何工作

模板入口是 `main.tex`。编译时 LaTeX 从 `main.tex` 开始读取论文信息、生成封面和声明页，然后依次载入摘要、目录、正文、参考文献、附录、成果和致谢。

当前主文件的组织顺序如下：

```tex
\documentclass{ussthesis}

% 论文个人信息
\usstTitle{上海理工大学硕士学位论文 LaTeX 模板示例}
...

\begin{document}

\makeoutercover
\makeinnercover
\makeauthorization
\makedeclaration

\input{chapters/abstract}
\ussttableofcontents

\usstmainmatter
\include{chapters/chapter01}
\include{chapters/chapter02}
\include{chapters/conclusion}

\usstbibliography{references}

\usstappendix
\include{chapters/appendix}

\usstbackmatter
\include{chapters/publications}
\include{chapters/acknowledgements}

\end{document}
```

这些命令的含义如下：

| 命令 | 作用 |
| --- | --- |
| `\makeoutercover` | 生成外封面 |
| `\makeinnercover` | 生成中文、英文内封面 |
| `\makeauthorization` | 生成学位论文版权使用授权书 |
| `\makedeclaration` | 生成学位论文原创性声明 |
| `\input{chapters/abstract}` | 插入摘要文件 |
| `\ussttableofcontents` | 生成目录页 |
| `\usstmainmatter` | 切换到正文部分，正文页码从 1 开始 |
| `\include{chapters/chapter01}` | 插入一个章节文件 |
| `\usstbibliography{references}` | 插入 `references.bib` 对应的参考文献 |
| `\usstappendix` | 切换到附录部分 |
| `\usstbackmatter` | 切换到成果、致谢等正文之后的部分 |

## 2. 安装软件和配置环境

### 2.1 必须安装的软件

使用本模板至少需要以下软件：

| 软件 | 用途 | 建议 |
| --- | --- | --- |
| TeX Live | 提供 XeLaTeX、BibTeX、latexmk 和宏包 | 推荐 TeX Live 2026 或较新的完整版安装 |
| VSCode | 编辑 `.tex`、`.bib` 和 Markdown 文件 | 推荐使用稳定版 |
| LaTeX Workshop | VSCode 中的 LaTeX 编译、预览和清理扩展 | 必装 |

本模板使用 XeLaTeX 编译，原因是中文论文需要稳定处理宋体、黑体、楷体等系统字体。不要使用 pdfLaTeX 编译本模板。

### 2.2 推荐安装的软件

以下软件不是模板编译的硬性要求，但写论文时很实用：

| 软件 | 用途 |
| --- | --- |
| Git | 版本管理，防止误删或覆盖论文内容 |
| Zotero | 管理文献，并导出 BibTeX 条目 |
| SumatraPDF | Windows 上轻量 PDF 阅读器，可配合 VSCode 正反向搜索 |
| Inkscape | 将 SVG、PDF、EPS 等矢量图转换为 PDF 或 PNG |
| ImageMagick | 批量转换图片格式，例如把 `.tif` 转为 `.png` |

如果只是正常写作，先安装 TeX Live、VSCode 和 LaTeX Workshop 就可以开始。

### 2.3 安装 TeX Live

建议安装 TeX Live 完整版。安装时注意：

1. 选择完整安装方案，不要只安装最小集合，否则可能缺少 `ctex`、`xeCJK`、`gbt7714`、`latexmk` 等宏包。
2. 安装路径尽量不要包含中文或空格，例如 `C:\texlive\2026`。
3. 勾选将 TeX Live 加入系统 `PATH`，或安装后手动把 `C:\texlive\2026\bin\windows` 加入 `PATH`。
4. 同一台机器上尽量不要混用多个 LaTeX 发行版，例如 TeX Live、MiKTeX、TinyTeX、SimpleTeX 同时存在时，容易出现命令路径混乱。

安装完成后，重新打开 PowerShell，运行：

```powershell
xelatex --version
bibtex --version
latexmk --version
kpsewhich ctexbook.cls
kpsewhich gbt7714-numeric.bst
```

能看到版本信息，并且 `kpsewhich` 能找到对应文件，就说明基础工具链可用。

若命令行出现 TeX Live Windows `runscript.tlu` 层面的环境变量错误，可先在当前 PowerShell 中执行：

```powershell
$env:WINDIR='C:\Windows'
$env:SystemRoot='C:\Windows'
```

然后再运行编译命令。

### 2.4 检查 PATH 是否正确

在 PowerShell 中运行：

```powershell
where.exe xelatex
where.exe bibtex
where.exe latexmk
```

正常情况下应指向 TeX Live 的 `bin\windows` 目录，例如：

```text
C:\texlive\2026\bin\windows\xelatex.exe
C:\texlive\2026\bin\windows\bibtex.exe
C:\texlive\2026\bin\windows\latexmk.exe
```

如果结果指向 SimpleTeX、MiKTeX 或旧版 TeX Live，而你希望使用 TeX Live 2026，需要调整系统环境变量中的 `PATH` 顺序，把 `C:\texlive\2026\bin\windows` 放到其他 LaTeX 路径前面。调整后关闭并重新打开 VSCode 和 PowerShell。

### 2.5 配置 VSCode

1. 安装 VSCode。
2. 在 VSCode 扩展市场安装 `LaTeX Workshop`。
3. 用 VSCode 打开整个模板文件夹，不要只打开 `main.tex`。
4. 确认左侧文件树中能看到 `.vscode/settings.json`。
5. 打开 `main.tex`，点击 VSCode 左侧 TeX 图标。
6. 在 `Build LaTeX project` 中选择 `Recipe: XeLaTeX -> BibTeX -> XeLaTeX*2`。

当前工程的 `.vscode/settings.json` 已经写好以下工具链：

```json
{
  "latex-workshop.latex.recipe.default": "first",
  "latex-workshop.latex.tools": [
    {
      "name": "xelatex",
      "command": "xelatex",
      "args": [
        "-synctex=1",
        "-interaction=nonstopmode",
        "-file-line-error",
        "%DOC%"
      ]
    },
    {
      "name": "bibtex",
      "command": "bibtex",
      "args": [
        "%DOCFILE%"
      ]
    }
  ],
  "latex-workshop.latex.recipes": [
    {
      "name": "XeLaTeX -> BibTeX -> XeLaTeX*2",
      "tools": [
        "xelatex",
        "bibtex",
        "xelatex",
        "xelatex"
      ]
    }
  ]
}
```

如果 VSCode 编译失败，但 PowerShell 中可以正常编译，通常是 VSCode 没有继承最新的系统环境变量。关闭所有 VSCode 窗口后重新打开即可。

### 2.6 配置字体

模板在 Windows 下默认使用以下字体：

| 字体 | 用途 |
| --- | --- |
| SimSun | 中文正文宋体 |
| SimHei | 中文黑体标题 |
| FangSong | 仿宋相关文字 |
| KaiTi | 楷体相关文字 |
| Times New Roman | 英文和数字 |
| Arial | 无衬线英文字体 |
| Consolas | 等宽字体 |

Windows 系统通常自带这些字体。可在 PowerShell 中简单检查：

```powershell
Get-ChildItem 'C:\Windows\Fonts' | Where-Object { $_.Name -match 'simsun|simhei|simfang|simkai|times|arial|consola' } | Select-Object Name
```

如果缺少字体，PDF 中可能出现字体替换、粗细异常或编译警告。优先在系统中安装对应字体，而不是直接修改模板字体设置。

### 2.7 配置文献管理工具

如果使用 Zotero 管理文献，建议安装 Better BibTeX 插件，并按以下方式使用：

1. 在 Zotero 中维护文献库。
2. 为每篇文献设置稳定的 citation key，例如 `zhang2026model`。
3. 导出 BibTeX，或复制单条 BibTeX 到 `references.bib`。
4. 确认作者字段用 `and` 分隔。
5. 在正文中使用 `\cite{zhang2026model}`。

不使用 Zotero 也可以，直接从论文网站、Google Scholar、DBLP、CNKI 或出版社页面复制 BibTeX 条目到 `references.bib`。

### 2.8 安装后做一次完整验证

软件装好后，在模板文件夹中执行：

```powershell
$env:WINDIR='C:\Windows'
$env:SystemRoot='C:\Windows'
latexmk -xelatex -interaction=nonstopmode -file-line-error main.tex
```

如果生成 `main.pdf`，说明模板可以正常编译。然后再检查参考文献是否显示、目录是否更新、图表编号是否正常。

## 3. 编译模板

### 3.1 使用 VSCode 编译

推荐用 VSCode 打开整个文件夹，而不是只打开单个 `.tex` 文件。当前工程已经提供 `.vscode/settings.json`，LaTeX Workshop 中可使用（默认流程）：

```text
XeLaTeX -> BibTeX -> XeLaTeX*2
```

这套流程会先生成正文，再处理参考文献，最后再编译两次更新引用、目录和页码。
也可通过窗口右上角按钮或快捷键（Ctrl + Alt +B）一键构建。

### 3.2 使用命令行编译

推荐命令：

```powershell
$env:WINDIR='C:\Windows'
$env:SystemRoot='C:\Windows'
latexmk -xelatex -interaction=nonstopmode -file-line-error main.tex
```

如果不用 `latexmk`，手动执行：

```powershell
xelatex -synctex=1 -interaction=nonstopmode -file-line-error main.tex
bibtex main
xelatex -synctex=1 -interaction=nonstopmode -file-line-error main.tex
xelatex -synctex=1 -interaction=nonstopmode -file-line-error main.tex
```

第一次编译后如果目录、交叉引用或参考文献显示为 `??`，通常不是正文错误，而是还需要完整执行 BibTeX 和后续两次 XeLaTeX。

### 3.3 清理辅助文件

编译会产生 `.aux`、`.bbl`、`.blg`、`.log`、`.out`、`.toc`、`.synctex.gz`、`.fls`、`.fdb_latexmk` 等辅助文件。一般不需要手动删除。若要重新干净编译，可在 VSCode LaTeX Workshop 中使用清理功能，或执行：

```powershell
latexmk -C main.tex
```

清理后再重新运行完整编译命令。

## 4. 修改个人信息

个人信息集中在 `main.tex` 顶部。直接改花括号里的内容即可。

```tex
\usstSchoolCode{10252}
\usstStudentID{XXXXXXXXX}
\usstClassification{TP391（查询后调整）}
\usstDegree{硕士}
\usstDegreeEn{Master}
\usstTitle{上海理工大学硕士学位论文 LaTeX 模板示例}
\usstTitleEn{A LaTeX Template for Master's Thesis of University of Shanghai for Science and Technology}
\usstAuthor{张三}
\usstAuthorEn{Zhang San}
\usstDepartment{学院名称}
\usstDiscipline{专业名称}
\usstResearchField{研究方向}
\usstSupervisor{李四\quad 教授（根据实际调整）}
\usstSupervisorEn{Professor Li Si}
\usstDate{二〇二七年四月}
\usstDateArabic{2027年4月}
\usstDateEn{April 2027}
\usstDegreeMode{全日制}
```

各字段说明：

| 命令 | 填写内容 |
| --- | --- |
| `\usstSchoolCode{...}` | 学校代码，上海理工大学为 `10252` |
| `\usstStudentID{...}` | 学号 |
| `\usstClassification{...}` | 中图分类号或学院要求填写的分类号 |
| `\usstDegree{...}` | 中文学位层次，硕士论文通常填 `硕士` |
| `\usstDegreeEn{...}` | 英文学位层次，硕士通常填 `Master` |
| `\usstTitle{...}` | 中文论文题目 |
| `\usstTitleEn{...}` | 英文论文题目 |
| `\usstAuthor{...}` | 作者中文姓名 |
| `\usstAuthorEn{...}` | 作者英文或拼音姓名 |
| `\usstDepartment{...}` | 学院名称 |
| `\usstDiscipline{...}` | 专业名称、学科或专业学位类别 |
| `\usstResearchField{...}` | 研究方向 |
| `\usstSupervisor{...}` | 导师姓名和职称，可用 `\quad` 拉开间距 |
| `\usstSupervisorEn{...}` | 导师英文姓名和职称 |
| `\usstDate{...}` | 外封面中文大写日期，例如 `二〇二七年四月` |
| `\usstDateArabic{...}` | 中文内封面阿拉伯数字日期，例如 `2027年4月` |
| `\usstDateEn{...}` | 英文内封面日期，例如 `April 2027` |
| `\usstDegreeMode{...}` | 学习方式，例如 `全日制` |

如果论文题目很长，可以直接写在同一个命令中，LaTeX 会自动换行。不要手动插入大量空格。确实需要强制换行时，可谨慎使用 `\\`，但封面题名强制换行可能影响对齐，建议先自动换行。

## 5. 修改摘要和关键词

摘要文件是 `chapters/abstract.tex`。中文摘要放在 `cnabstract` 环境中，英文摘要放在 `enabstract` 环境中。

```tex
\begin{cnabstract}
这里填写中文摘要正文。摘要通常说明研究背景、研究目的、研究方法、主要结果和结论。

\cnkeywords{关键词一\quad 关键词二\quad 关键词三}
\end{cnabstract}

\begin{enabstract}
Write the English abstract here.

\enkeywords{keyword one, keyword two, keyword three}
\end{enabstract}
```

关键词写法：

- 中文关键词使用 `\cnkeywords{...}`。
- 中文关键词之间建议用 `\quad` 分隔。
- 英文关键词使用 `\enkeywords{...}`。
- 英文关键词之间可用英文逗号和空格分隔。

当前模板会在目录页第一章之前直接打印：

```text
中文摘要
ABSTRACT
```

这两项不带页码。如果学院要求不显示这两项，可在 `main.tex` 中取消下面这一行的注释：

```tex
\usstAbstractNotInToc
```

## 6. 输入正文内容

正文文件放在 `chapters/` 文件夹中。每个章节文件通常以 `\chapter{...}` 开头。

一个最小章节示例：

```tex
\chapter{引言}

\section{研究背景}

这里写正文第一段。LaTeX 中普通换行不会开始新段落，空一行才会开始新段落。

这里写正文第二段。

\section{研究内容}

这里继续写正文。
```

正文写作注意事项：

- 段落之间空一行。
- 中文正文不需要手动在段首输入空格，模板会自动首行缩进。
- 英文单词之间正常输入空格。
- 百分号 `%` 是注释符号，正文中要显示百分号请写 `\%`。
- 下划线 `_` 在正文中有特殊含义，普通文本要显示下划线请写 `\_`。
- 反斜杠 `\` 是 LaTeX 命令起始符，不要随意输入。
- 文件名建议使用英文、数字和连字符，例如 `chapter03-method.tex`，可以减少路径兼容问题。

## 7. 增加、删除和调整章节

### 7.1 增加一章

例如要新增第三章“方法设计”，可以新建文件：

```text
chapters/chapter03-method.tex
```

文件内容示例：

```tex
\chapter{方法设计}

\section{总体框架}

这里写第三章内容。

\section{关键算法}

这里写算法或方法细节。
```

然后在 `main.tex` 的正文区域加入：

```tex
\usstmainmatter
\include{chapters/chapter01}
\include{chapters/chapter02}
\include{chapters/chapter03-method}
\include{chapters/conclusion}
```

不需要写 `.tex` 后缀。章节编号会自动更新。

### 7.2 删除一章

如果不需要某章，在 `main.tex` 中删除或注释对应的 `\include{...}` 即可。

```tex
% \include{chapters/chapter02}
```

章节文件可以保留在 `chapters/` 中作为备份，只要没有被 `main.tex` 引入，就不会出现在论文里。

### 7.3 调整章节顺序

调整 `main.tex` 中 `\include{...}` 的顺序即可。例如：

```tex
\include{chapters/chapter01}
\include{chapters/chapter03-method}
\include{chapters/chapter02}
\include{chapters/conclusion}
```

LaTeX 会自动重排章号、图号、表号、公式编号和目录。

### 7.4 章节首页

模板已设置正文每一章首页从奇数页开始。如果前一章最后一页后需要插入空白页，模板会保留正文页眉页脚。

## 8. 增加和删除节标题

正文标题层级建议使用到二级节标题：

```tex
\chapter{章标题}

\section{一级节标题}

\subsection{二级节标题}
```

模板也支持三级节标题：

```tex
\subsubsection{三级节标题}
```

但学校规范中无特殊情况不建议使用三级及以上节标题，因此正文中尽量控制在 `\section` 和 `\subsection`。

删除节标题时，直接删除对应的标题命令和该小节内容即可。标题编号会自动更新，不要手工写“1.1”“1.2.1”等编号。

正确写法：

```tex
\section{研究背景}
```

不推荐写法：

```tex
\section{1.1 研究背景}
```

## 9. 插入图片

### 9.1 图片放在哪里

图片统一放在 `figures/` 文件夹中。当前示例图是：

```text
figures/fig1.png
```

正文插图建议使用 `.png`、`.jpg` 或 `.pdf`，不要直接插入 `.tif`。如果原始图片是 `.tif`，建议先转换成 `.png` 或 `.pdf` 后放入 `figures/`。如果图片来自截图或位图，推荐导出为 `.png`；如果图片来自矢量图、Matlab、Python、Origin 或 Illustrator，推荐导出为 `.pdf`。

### 9.2 最小图片示例

在正文中插入图片：

```tex
\begin{figure}[htbp]
  \centering
  \includegraphics[width=0.8\textwidth]{figures/fig1.png}
  \caption{示例图}
  \label{fig:demo}
\end{figure}
```

说明：

| 写法 | 含义 |
| --- | --- |
| `figure` | 图片浮动体环境 |
| `[htbp]` | 允许图片放在当前位置、页顶、页底或单独浮动页 |
| `\centering` | 图片居中 |
| `\includegraphics{...}` | 插入图片文件 |
| `width=0.8\textwidth` | 图片宽度为正文宽度的 80% |
| `\caption{...}` | 图题 |
| `\label{fig:demo}` | 交叉引用标签 |

图题会自动排在图下方，图号会按章编号，例如“图 2.1”。

### 9.3 调整图片大小

常用写法：

```tex
\includegraphics[width=0.6\textwidth]{figures/my-figure.png}
\includegraphics[width=\textwidth]{figures/my-wide-figure.png}
\includegraphics[height=6cm]{figures/my-figure.png}
```

不要同时随意设置 `width` 和 `height`，否则图片可能被拉伸变形。若确实需要限制宽高，可以加 `keepaspectratio`：

```tex
\includegraphics[width=\textwidth,height=8cm,keepaspectratio]{figures/my-figure.png}
```

如果想按图片原始尺寸插入，可以不写宽度：

```tex
\includegraphics{figures/my-figure.png}
```

但论文中通常更推荐按 `\textwidth` 的比例控制图片宽度，因为原始像素尺寸不等于论文中的物理显示尺寸。

### 9.4 引用图片

图片的 `\label` 建议放在 `\caption` 后面。正文引用时写：

```tex
如图~\ref{fig:demo} 所示，系统由三个模块组成。
```

标签命名建议：

```text
fig:framework
fig:experiment-result
fig:ablation
```

注意：每个 `\label` 必须唯一，不能重复。

## 10. 插入和填写表格

模板已加载 `booktabs`，推荐使用三线表。

### 10.1 基础三线表

```tex
\begin{table}[htbp]
  \centering
  \caption{模板中的三线表示例}
  \label{tab:demo}
  \begin{tabular}{ccc}
    \toprule
    指标 & 方法一 & 方法二 \\
    \midrule
    准确率 & 91.2\% & 94.5\% \\
    时间/s & 12.4 & 9.8 \\
    \bottomrule
  \end{tabular}
\end{table}
```

说明：

| 写法 | 含义 |
| --- | --- |
| `table` | 表格浮动体环境 |
| `\caption{...}` | 表题，模板会放在表格上方 |
| `\label{tab:demo}` | 表格引用标签 |
| `tabular` | 真正的表格内容 |
| `{ccc}` | 三列，均居中 |
| `\toprule` | 顶线 |
| `\midrule` | 中线 |
| `\bottomrule` | 底线 |
| `&` | 分隔列 |
| `\\` | 换行 |

表号会按章编号，例如“表 2.1”。

### 10.2 列对齐方式

`tabular` 的列格式常用：

| 符号 | 含义 |
| --- | --- |
| `l` | 左对齐 |
| `c` | 居中 |
| `r` | 右对齐 |
| `p{3cm}` | 固定宽度列，文字可自动换行 |

示例：

```tex
\begin{tabular}{lcr}
  \toprule
  名称 & 数量 & 备注 \\
  \midrule
  样本 A & 100 & 正常 \\
  样本 B & 80 & 异常 \\
  \bottomrule
\end{tabular}
```

固定宽度列示例：

```tex
\begin{tabular}{lp{8cm}}
  \toprule
  项目 & 说明 \\
  \midrule
  方法 & 这里是一段较长的说明文字，会在固定宽度内自动换行。 \\
  \bottomrule
\end{tabular}
```

### 10.3 使用 `tabularx` 自动撑满正文宽度

模板已加载 `tabularx`，适合说明文字较多的表格：

```tex
\begin{table}[htbp]
  \centering
  \caption{模块说明}
  \label{tab:modules}
  \begin{tabularx}{\textwidth}{lX}
    \toprule
    模块 & 功能说明 \\
    \midrule
    数据采集 & 负责采集实验数据并进行初步清洗。 \\
    模型训练 & 负责训练预测模型并输出评价指标。 \\
    \bottomrule
  \end{tabularx}
\end{table}
```

其中 `X` 列会自动占据剩余宽度。

### 10.4 表格中的特殊字符

表格中常见特殊字符写法：

| 想显示 | LaTeX 写法 |
| --- | --- |
| `%` | `\%` |
| `_` | `\_` |
| `&` | `\&` |
| `#` | `\#` |

如果表格列数不一致，常见报错是 `Extra alignment tab has been changed to \cr`。检查每一行的 `&` 数量是否一致。

### 10.5 引用表格

正文引用：

```tex
实验参数如表~\ref{tab:demo} 所示。
```

表格标签建议使用 `tab:` 前缀，例如：

```text
tab:parameters
tab:dataset
tab:comparison
```

## 11. 插入公式

模板已加载 `amsmath` 和 `amssymb`，可直接输入常用数学公式。

### 11.1 行内公式

行内公式写在 `$...$` 中：

```tex
当样本数量为 $n$ 时，平均值记为 $\bar{x}$。
```

### 11.2 单行编号公式

```tex
\begin{equation}
  E = mc^2
  \label{eq:energy}
\end{equation}
```

正文引用：

```tex
式~\eqref{eq:energy} 给出了能量与质量之间的关系。
```

公式编号会按章编号，例如 `(2.1)`。

### 11.3 多行公式

多行对齐公式用 `align`：

```tex
\begin{align}
  y &= ax + b, \\
  z &= cx + d .
  \label{eq:linear-model}
\end{align}
```

`&` 表示对齐位置，通常放在等号前。若某一行不需要编号，可在该行末尾使用 `\notag`：

```tex
\begin{align}
  y &= ax + b, \notag \\
  z &= cx + d .
  \label{eq:linear-model}
\end{align}
```

### 11.4 分段或条件公式

```tex
\begin{equation}
  f(x)=
  \begin{cases}
    0, & x < 0, \\
    x, & x \ge 0 .
  \end{cases}
  \label{eq:relu}
\end{equation}
```

### 11.5 公式写作建议

- 变量第一次出现时应解释含义。
- 单位尽量写在正文中，不要和变量混在一起导致歧义。
- 公式末尾可以按句子需要加逗号或句号。
- 引用公式时用 `\eqref{...}`，不要手工输入公式编号。

## 12. 导入和插入参考文献

参考文献数据库是 `references.bib`。正文中使用 `\cite{...}` 引用，参考文献列表由 `\usstbibliography{references}` 自动生成。

### 12.1 BibTeX 条目结构

一个期刊论文条目示例：

```bibtex
@article{zhang2024article,
  author  = {张三 and 李四 and 王五},
  title   = {学术论文参考文献格式示例},
  journal = {上海理工大学学报},
  year    = {2024},
  volume  = {46},
  number  = {2},
  pages   = {101--108}
}
```

其中 `zhang2024article` 是文献键。正文引用时写：

```tex
相关研究已有较多报道\cite{zhang2024article}。
```

多篇文献一起引用：

```tex
相关格式可参考已有文献\cite{gbt7714-2015,lamport1994latex}。
```

### 12.2 常见参考文献类型

中文或英文期刊论文：

```bibtex
@article{key2026article,
  author  = {作者一 and 作者二},
  title   = {论文题名},
  journal = {期刊名称},
  year    = {2026},
  volume  = {10},
  number  = {2},
  pages   = {1--10}
}
```

会议论文：

```bibtex
@inproceedings{key2026conference,
  author    = {作者一 and 作者二},
  title     = {会议论文题名},
  booktitle = {会议论文集名称},
  address   = {会议地点},
  year      = {2026},
  pages     = {20--30}
}
```

专著：

```bibtex
@book{key2026book,
  author    = {作者},
  title     = {书名},
  edition   = {2},
  address   = {出版地},
  publisher = {出版社},
  year      = {2026}
}
```

硕士学位论文：

```bibtex
@mastersthesis{key2026master,
  author  = {作者},
  title   = {硕士学位论文题名},
  address = {城市},
  school  = {学校名称},
  year    = {2026}
}
```

博士学位论文：

```bibtex
@phdthesis{key2026phd,
  author  = {作者},
  title   = {博士学位论文题名},
  address = {城市},
  school  = {学校名称},
  year    = {2026}
}
```

标准：

```bibtex
@standard{key2026standard,
  author    = {{发布机构}},
  title     = {标准名称},
  number    = {标准编号},
  address   = {出版地},
  publisher = {出版社},
  year      = {2026}
}
```

### 12.3 从 Zotero、EndNote 或网页导入

常见流程：

1. 在 Zotero、EndNote、Google Scholar、DBLP、CNKI 或出版社页面导出 BibTeX。
2. 将导出的条目复制到 `references.bib`。
3. 检查文献键是否唯一，例如 `smith2024model`。
4. 检查作者之间是否用 `and` 分隔。
5. 检查英文题名中需要保留大写的词是否用花括号保护，例如 `{USST}`、`{CNN}`、`{English}`。
6. 在正文中用 `\cite{文献键}` 引用。
7. 重新执行完整编译链。

作者字段示例：

```bibtex
author = {Smith, John and Zhang, San and 李四}
```

不要用中文顿号、逗号或分号分隔 BibTeX 作者。

### 12.4 参考文献不显示怎么办

检查顺序：

1. 正文是否真的写了 `\cite{key}`。
2. `references.bib` 中是否存在完全相同的 `key`。
3. 是否运行了 BibTeX。
4. 是否在 BibTeX 后又运行了两次 XeLaTeX。
5. `references.bib` 是否有漏逗号、漏右花括号、重复文献键。

## 15. 目录和页码

模板已经处理好第一章奇数页开章、页眉页脚和页码切换。正文前的封面、声明、摘要、ABSTRACT 和目录是否插入空白背面页，以及第二章、第三章和后续各章是否强制奇数页开章，都可以手动选择。

一般不要手动修改以下命令顺序：

```tex
\input{chapters/abstract}
\ussttableofcontents

\usstmainmatter
```

当前目录页会在第一章之前打印“中文摘要”和“ABSTRACT”，并且这两项不标页码。如果不需要，启用：

```tex
\usstAbstractNotInToc
```

该命令应放在 `main.tex` 顶部论文信息之后、`\begin{document}` 之前。

### 15.1 前置页空白背面开关

默认情况下，模板不在正文前的前置页之间插入空白背面页。这适合提交电子版 PDF，文件页数更少，阅读和跳转也更直接。

如果需要导出打印版本，并希望外封面、内封面、授权书、声明、摘要、ABSTRACT、目录等正文前页面按单面打印效果保留背面空白页，可在 `main.tex` 顶部论文信息之后、`\begin{document}` 之前取消下面这一行的注释：

```tex
\usstFrontMatterBlankPages
```

当前 `main.tex` 中已经预留了这个位置：

```tex
% Electronic version: keep front-matter blank backs disabled by default.
% Print version: uncomment the next line to insert blank backs before Chapter 1.
% \usstFrontMatterBlankPages
```

如果之后想恢复电子版模式，把这一行重新注释掉即可：

```tex
% \usstFrontMatterBlankPages
```

也可以显式写成：

```tex
\usstFrontMatterNoBlankPages
```

无论是否开启前置页空白背面，模板都会在 `\usstmainmatter` 进入正文前检查实际 PDF 页序，必要时自动补一页空白页，保证第一章首页位于奇数页。

### 15.2 正文各章奇数页开章开关

第一章首页始终会保持在奇数页，这一点不受本开关影响。

默认情况下，模板会在第二章、第三章和后续各章前按需插入空白页，保证每一章首页都位于奇数页。默认设置等同于：

```tex
\usstChapterOddPages
```

如果认为学校规范没有要求第二章及后续各章必须从奇数页开始，并希望减少正文中的空白页，可在 `main.tex` 顶部论文信息之后、`\begin{document}` 之前取消下面这一行的注释：

```tex
\usstChapterAnyPage
```

当前 `main.tex` 中已经预留了这个位置：

```tex
% Chapter 2 and later start on odd pages by default.
% To avoid inserted blank pages between chapters, uncomment:
% \usstChapterAnyPage
```

如果之后想恢复各章奇数页开章，把这一行重新注释掉即可；也可以显式写成：

```tex
\usstChapterOddPages
```

## 16. 常见问题

### 16.1 图片大小为什么和原始大小不一致

LaTeX 输出的是论文页面中的物理尺寸，不是图片文件的像素尺寸。下面的写法会强制图片显示为正文宽度的 80%：

```tex
\includegraphics[width=0.8\textwidth]{figures/fig1.png}
```

如果想变小，改成 `0.6\textwidth`；如果想铺满正文宽度，改成 `\textwidth`。

### 16.2 引用显示为问号

常见原因是没有完整编译。运行：

```powershell
xelatex main.tex
bibtex main
xelatex main.tex
xelatex main.tex
```

或直接运行：

```powershell
latexmk -xelatex main.tex
```

### 16.3 参考文献没有出现在列表里

只有正文中被 `\cite{...}` 引用过的文献才会出现在参考文献列表中。如果希望某篇文献进入列表，必须在正文合适位置引用。

### 16.4 编译提示 `xeCJK Warning: Redefining CJKfamily`

这是当前模板和 `ctex` 字体设置共同导致的重复定义提示。只要 PDF 正常生成、字体显示正常，一般不影响使用。

### 16.5 表格报错 `Extra alignment tab`

这通常是某一行的 `&` 数量和列格式不一致。例如 `{ccc}` 表示三列，每一行应有两个 `&`。

### 16.6 中文路径会不会有问题

当前工程可以放在中文路径下编译。但图片文件名、章节文件名和 BibTeX 文献键仍建议优先使用英文、数字和连字符，减少跨系统协作时的兼容问题。

## 17. 推荐写作流程

1. 先在 `main.tex` 填好个人信息。
2. 在 `chapters/abstract.tex` 写摘要和关键词。
3. 按章节在 `chapters/` 中新建或修改 `.tex` 文件。
4. 每写完一节就编译一次，及时发现语法错误。
5. 图片统一放入 `figures/`，正文用 `figure` 环境插入。
6. 表格优先使用 `booktabs` 三线表。
7. 参考文献随写随加到 `references.bib`，正文用 `\cite{...}` 引用。
8. 初稿完成后执行一次完整编译，检查目录、页码、图表编号、公式编号和参考文献。
9. 提交前通读 `main.pdf`，重点检查封面信息、摘要、目录、每章首页、图表位置和参考文献格式。
